# Contributing to Smart Classroom Watch

First off, thank you for considering contributing to Smart Classroom Watch! It's people like you that make this project great.

## How Can I Contribute?

### Reporting Bugs
- Use the issue tracker
- Describe the bug clearly
- Include steps to reproduce
- Specify your environment (OS, versions, etc.)

### Suggesting Enhancements
- Open an issue with the "enhancement" label
- Provide a clear description of the feature
- Explain why this enhancement would be useful

### Pull Requests
1. Fork the repo
2. Create a new branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Write or update tests
5. Ensure code passes all tests
6. Commit with clear messages
7. Push to your fork
8. Submit a pull request

## Code Style

### C/C++ (Firmware)
- Follow Arduino style guide
- Use meaningful variable names
- Comment complex logic

### JavaScript/Node.js
- Use ES6+ features
- Follow Airbnb style guide
- Use semicolons
- 2-space indentation

### Python
- Follow PEP 8
- Use type hints where appropriate
- Document functions with docstrings

## Testing
- Write unit tests for new features
- Ensure all tests pass before submitting PR
- Maintain or improve code coverage

## Documentation
- Update README.md if needed
- Document new features in docs/
- Add inline comments for complex code

## Questions?
Feel free to open an issue or contact the maintainers.
